# 5.3 - Détection de Piétons (CCTV)

## 🎯 Concept

La détection de piétons en temps réel à partir d'une caméra de surveillance (CCTV) utilise des classificateurs Haar Cascade spécifiquement entraînés pour reconnaître les silhouettes humaines. Cette technologie est largement utilisée dans les systèmes de surveillance intelligente.

### Points clés
- **Traitement temps réel** : Analyse en direct du flux vidéo
- **Classificateur fullbody** : `haarcascade_fullbody.xml` détecte le corps entier
- **Classificateur upperbody** : `haarcascade_upperbody.xml` détecte le haut du corps (visage + torse)
- **Optimisation CCTV** : Paramètres adaptés aux caméras fixes avec éclairage variable

---

## ⚠️ IMPORTANT : picamera2 vs cv2.VideoCapture

### Pourquoi on utilise picamera2 sur Raspberry Pi 5 ?

Sur **Raspberry Pi 5 avec caméra CSI**, `cv2.VideoCapture(0)` **ne fonctionne PAS** à cause d'un bug avec le backend GStreamer. La solution est d'utiliser **picamera2** pour capturer, puis OpenCV pour le traitement d'image.

### Différences clés

| Aspect | cv2.VideoCapture | picamera2 |
|--------|------------------|-----------|
| **Plateforme** | PC / Webcam USB | Raspberry Pi + CSI |
| **Initialisation** | `cv2.VideoCapture(0)` | `Picamera2()` + `configure()` |
| **Capture** | `capture.read()` | `picam2.capture_array()` |
| **Format couleur** | BGR (OpenCV natif) | RGB (conversion nécessaire) |
| **Arrêt** | `capture.release()` | `picam2.stop()` |

### Conversion obligatoire RGB → BGR

```python
# picamera2 capture en RGB, OpenCV travaille en BGR
frame = picam2.capture_array()           # Format RGB
frame = cv2.cvtColor(frame, cv2.COLOR_RGB2BGR)  # Conversion BGR
```

Sans cette conversion, les couleurs seront inversées (rouge ↔ bleu) !

## 💻 Code Minimal

### Version Standard (PC/Webcam USB)

```python
import cv2
import time

# Chargement des classificateurs
body_detect = cv2.CascadeClassifier('haarcascade_fullbody.xml')
upperbody_detect = cv2.CascadeClassifier('haarcascade_upperbody.xml')

# Ouverture de la source vidéo
capture = cv2.VideoCapture(0)  # ou 'video.mp4' pour un fichier

while capture.isOpened():
    ret, frame = capture.read()
    if not ret:
        break
    
    gray = cv2.cvtColor(frame, cv2.COLOR_BGR2GRAY)
    
    # Détection corps entier
    bodies = body_detect.detectMultiScale(gray, 1.1, 3)
    
    # Détection haut du corps
    upper_bodies = upperbody_detect.detectMultiScale(gray, 1.1, 3)
    
    # Dessin des rectangles
    for (x, y, w, h) in bodies:
        cv2.rectangle(frame, (x, y), (x+w, y+h), (0, 255, 0), 2)
        cv2.putText(frame, 'Personne', (x, y-10), 
                   cv2.FONT_HERSHEY_SIMPLEX, 0.5, (0, 255, 0), 2)
    
    for (x, y, w, h) in upper_bodies:
        cv2.rectangle(frame, (x, y), (x+w, y+h), (255, 0, 0), 2)
        cv2.putText(frame, 'Haut corps', (x, y-10),
                   cv2.FONT_HERSHEY_SIMPLEX, 0.5, (255, 0, 0), 2)
    
    cv2.imshow('Détection Piétons CCTV', frame)
    
    if cv2.waitKey(1) == 27:
        break

capture.release()
cv2.destroyAllWindows()
```

### Version Raspberry Pi 5 + Caméra CSI (picamera2)

```python
import cv2
import time
from picamera2 import Picamera2

# Chargement des classificateurs
body_detect = cv2.CascadeClassifier('haarcascade_fullbody.xml')
upperbody_detect = cv2.CascadeClassifier('haarcascade_upperbody.xml')

# Initialisation picamera2
picam2 = Picamera2()
picam2.configure(picam2.create_preview_configuration(
    main={"format": "RGB888", "size": (640, 480)}
))
picam2.start()
time.sleep(0.1)

while True:
    # Capture et conversion
    frame = picam2.capture_array()
    frame = cv2.cvtColor(frame, cv2.COLOR_RGB2BGR)
    
    gray = cv2.cvtColor(frame, cv2.COLOR_BGR2GRAY)
    
    # Détection corps entier
    bodies = body_detect.detectMultiScale(gray, 1.1, 3)
    
    # Détection haut du corps
    upper_bodies = upperbody_detect.detectMultiScale(gray, 1.1, 3)
    
    # Dessin des rectangles
    for (x, y, w, h) in bodies:
        cv2.rectangle(frame, (x, y), (x+w, y+h), (0, 255, 0), 2)
        cv2.putText(frame, 'Personne', (x, y-10), 
                   cv2.FONT_HERSHEY_SIMPLEX, 0.5, (0, 255, 0), 2)
    
    for (x, y, w, h) in upper_bodies:
        cv2.rectangle(frame, (x, y), (x+w, y+h), (255, 0, 0), 2)
        cv2.putText(frame, 'Haut corps', (x, y-10),
                   cv2.FONT_HERSHEY_SIMPLEX, 0.5, (255, 0, 0), 2)
    
    cv2.imshow('Détection Piétons CCTV', frame)
    
    if cv2.waitKey(1) == 27:
        break

picam2.stop()
cv2.destroyAllWindows()
```

## 🔧 Fonction détaillée

### `cv2.VideoCapture()` (PC / Webcam USB)
```python
capture = cv2.VideoCapture(0)       # Caméra par défaut
capture = cv2.VideoCapture('cctv.mp4')  # Fichier vidéo
```
- **Paramètre** : Index de caméra ou chemin fichier
- **Retour** : Objet VideoCapture

### `Picamera2()` (Raspberry Pi + Caméra CSI)
```python
from picamera2 import Picamera2

picam2 = Picamera2()
picam2.configure(picam2.create_preview_configuration(
    main={"format": "RGB888", "size": (640, 480)}
))
picam2.start()
```
- **RGB888** : Format 8 bits par canal RGB
- **Size** : Résolution (640x480 = rapide, 1920x1080 = lent)

### `picam2.capture_array()`
```python
frame = picam2.capture_array()
frame = cv2.cvtColor(frame, cv2.COLOR_RGB2BGR)  # Conversion !
```
- **Retour** : Image NumPy array (RGB)
- **Note** : Conversion BGR obligatoire pour OpenCV

## 🎨 Analogie Simple

Imagine un agent de sécurité dans une salle de surveillance :
- Il observe **plusieurs écrans** (caméras CCTV)
- Il utilise un **manuel de silhouettes** pour identifier les personnes
- Il peut différencier un **corps entier** d'un **haut de corps**
- Il entoure les personnes détectées avec un **marqueur fluorescent**
- Il note le **type** de détection (personne complète vs haut du corps)

## ⚠️ Pièges et Astuces

### ❌ Erreurs fréquentes

**1. Oublier la conversion RGB→BGR (picamera2)**
```python
# ❌ Mauvais - couleurs inversées
frame = picam2.capture_array()
cv2.imshow('CCTV', frame)  # Rouge ↔ Bleu inversés !

# ✅ Bon - conversion nécessaire
frame = picam2.capture_array()
frame = cv2.cvtColor(frame, cv2.COLOR_RGB2BGR)
cv2.imshow('CCTV', frame)  # Couleurs correctes
```

**2. Classificateurs non trouvés**
```python
# Les fichiers XML doivent être présents dans le dossier
# Télécharger depuis : https://github.com/opencv/opencv/tree/master/data/haarcascades
```

**3. Confusion fullbody vs upperbody**
- `fullbody` = silhouette complète (mieux pour personnes debout)
- `upperbody` = visage + torse (mieux pour personnes assises/partielles)

### 💡 Astuces

**Améliorer la détection en conditions difficiles**
```python
# Augmenter minNeighbors pour réduire les faux positifs
bodies = body_detect.detectMultiScale(gray, 1.2, 5)

# Diminuer scaleFactor pour détecter plus de tailles
bodies = body_detect.detectMultiScale(gray, 1.05, 3)
```

**Ajouter un compteur de personnes**
```python
count = len(bodies) + len(upper_bodies)
cv2.putText(frame, f'Personnes: {count}', (10, 30),
           cv2.FONT_HERSHEY_SIMPLEX, 1, (0, 255, 255), 2)
```

**Sauvegarder les détections**
```python
if len(bodies) > 0:
    cv2.imwrite(f'alerte_{time.time()}.jpg', frame)
```

## 🔗 Différences avec la détection de visages

| Aspect | Visages | Piétons |
|--------|---------|---------|
| **Classificateur** | `haarcascade_frontalface` | `haarcascade_fullbody` |
| **Objet détecté** | Visage uniquement | Corps entier ou partie |
| **Mouvement** | Peut être immobile | Souvent en mouvement |
| **Taille** | Relativement constante | Très variable (adulte/enfant) |
| **Contexte** | Frontal de préférence | Toutes orientations |

## 📝 Résumé

La détection de piétons en CCTV combine :
1. **Capture temps réel** avec picamera2 (RPi) ou VideoCapture (PC)
2. **Conversion RGB→BGR** obligatoire pour picamera2
3. **Détection multi-cascade** (fullbody + upperbody)
4. **Annotation visuelle** avec rectangles et labels
5. **Comptage/Alerte** selon le cas d'usage

**Point clé** : Sur Raspberry Pi 5, utiliser obligatoirement picamera2 avec conversion de couleur !
