# 5.1 - Détection Visage, Yeux et Nez en Temps Réel

## Concept
Capturer un flux vidéo depuis la caméra et détecter en temps réel les visages, yeux et nez humains à l'aide des **Haar Cascades** (classificateurs entraînés).

## Architecture du système

```
Caméra (0) → VideoCapture → Frame → Redimensionnement → Grayscale
                                                    ↓
                    ┌──────────────────────────────────────────┐
                    ↓                                          ↓
            Face Cascade                              ROI (visage détecté)
                    ↓                                          ↓
            Rectangle rouge                         Eye Cascade + Nose Cascade
                    ↓                                          ↓
            Affichage final ←────── Rectangles yeux (bleu) + nez (vert)
```

---

## ⚠️ IMPORTANT : picamera2 vs cv2.VideoCapture

### Pourquoi on utilise picamera2 sur Raspberry Pi 5 ?

Sur **Raspberry Pi 5 avec caméra CSI**, `cv2.VideoCapture(0)` **ne fonctionne PAS** à cause d'un bug avec le backend GStreamer. La solution est d'utiliser **picamera2** pour capturer, puis OpenCV pour le traitement d'image.

### Différences clés

| Aspect | cv2.VideoCapture | picamera2 |
|--------|------------------|-----------|
| **Plateforme** | PC / Webcam USB | Raspberry Pi + CSI |
| **Initialisation** | `cv2.VideoCapture(0)` | `Picamera2()` + `configure()` |
| **Capture** | `capture.read()` | `picam2.capture_array()` |
| **Format couleur** | BGR (OpenCV natif) | RGB (conversion nécessaire) |
| **Arrêt** | `capture.release()` | `picam2.stop()` |

### Conversion obligatoire RGB → BGR

```python
# picamera2 capture en RGB, OpenCV travaille en BGR
frame = picam2.capture_array()           # Format RGB
frame = cv2.cvtColor(frame, cv2.COLOR_RGB2BGR)  # Conversion BGR
```

Sans cette conversion, les couleurs seront inversées (rouge ↔ bleu) !

## 💻 Code Minimal

### Version Standard (PC/Webcam USB)

```python
import cv2

# 1. Charger les classificateurs Haar
face_cascade = cv2.CascadeClassifier('haarcascade_frontalface_alt.xml')
eye_cascade = cv2.CascadeClassifier('haarcascade_eye.xml')

# 2. Ouvrir la caméra (0 = caméra par défaut)
cap = cv2.VideoCapture(0)

while True:
    # 3. Capturer une frame
    ret, frame = cap.read()
    
    # 4. Convertir en niveaux de gris
    gray = cv2.cvtColor(frame, cv2.COLOR_BGR2GRAY)
    
    # 5. Détecter les visages
    faces = face_cascade.detectMultiScale(gray, 1.3, 5)
    
    # 6. Dessiner rectangles + détecter yeux dans chaque visage
    for (x, y, w, h) in faces:
        cv2.rectangle(frame, (x, y), (x+w, y+h), (0, 0, 255), 2)
        
        # ROI = Region Of Interest (zone du visage)
        roi_gray = gray[y:y+h, x:x+w]
        roi_color = frame[y:y+h, x:x+w]
        
        # Détecter yeux dans la ROI
        eyes = eye_cascade.detectMultiScale(roi_gray)
        for (ex, ey, ew, eh) in eyes:
            cv2.rectangle(roi_color, (ex, ey), (ex+ew, ey+eh), (255, 0, 0), 2)
    
    # 7. Afficher
    cv2.imshow('Detection', frame)
    
    # 8. Quitter avec ESC
    if cv2.waitKey(1) == 27:
        break

# 9. Libérer ressources
cap.release()
cv2.destroyAllWindows()
```

## Fonctions détaillées

### `cv2.VideoCapture(index)` (PC / Webcam USB)

```python
capture = cv2.VideoCapture(0)  # Caméra par défaut
capture = cv2.VideoCapture(2)  # 3ème caméra (0, 1, 2)
capture = cv2.VideoCapture('video.avi')  # Fichier vidéo
```

| Index | Usage |
|-------|-------|
| `0` ou `-1` | Caméra par défaut (USB interne) |
| `2` | Deuxième caméra externe |
| Chemin fichier | Lecture vidéo depuis fichier |

### `Picamera2()` (Raspberry Pi + Caméra CSI)

```python
from picamera2 import Picamera2

picam2 = Picamera2()
picam2.configure(picam2.create_preview_configuration(
    main={"format": "RGB888", "size": (640, 480)}
))
picam2.start()
```

| Paramètre | Description | Valeur typique |
|-----------|-------------|----------------|
| `format` | Format des pixels | `"RGB888"` = 8 bits RGB |
| `size` | Résolution | `(640, 480)` = VGA rapide |

### `picam2.capture_array()`

```python
frame = picam2.capture_array()  # Capture en RGB
frame = cv2.cvtColor(frame, cv2.COLOR_RGB2BGR)  # Conversion BGR
```

| Étape | Description |
|-------|-------------|
| Capture | `picam2.capture_array()` retourne RGB |
| Conversion | `cv2.cvtColor(..., COLOR_RGB2BGR)` obligatoire |
| Résultat | Image utilisable par OpenCV |

⚠️ **Sans conversion, les couleurs seront inversées !**

### `cv2.CascadeClassifier(fichier.xml)`

```python
face_detect = cv2.CascadeClassifier('haarcascade_frontalface_alt.xml')
```

Charge un modèle pré-entraîné pour la détection d'objets. Les fichiers `.xml` contiennent les caractéristiques apprises par le classificateur.

**Fichiers Haar Cascades courants :**

| Fichier | Détecte |
|---------|---------|
| `haarcascade_frontalface_default.xml` | Visage de face (standard) |
| `haarcascade_frontalface_alt.xml` | Visage de face (alternatif, souvent meilleur) |
| `haarcascade_eye.xml` | Yeux |
| `haarcascade_mcs_nose.xml` | Nez |
| `haarcascade_fullbody.xml` | Corps entier |
| `haarcascade_car.xml` | Voitures |

### `detectMultiScale(image, scaleFactor, minNeighbors)`

```python
detections = cascade.detectMultiScale(gray, scaleFactor=1.3, minNeighbors=5)
```

| Paramètre | Description | Valeur typique |
|-----------|-------------|----------------|
| `image` | Image en niveaux de gris | `gray` |
| `scaleFactor` | Réduction image entre scans (1.1 = 10%) | `1.1` à `1.5` |
| `minNeighbors` | Min de rectangles voisins pour valider | `3` à `6` |

**Retour :** Liste de tuples `(x, y, w, h)`
- `x, y` : Coin supérieur gauche
- `w, h` : Largeur et hauteur

### `cv2.rectangle(image, pt1, pt2, color, thickness)`

```python
cv2.rectangle(frame, (x, y), (x+w, y+h), (0, 0, 255), 2)
```

Dessine un rectangle sur l'image.

| Paramètre | Description | Exemple |
|-----------|-------------|---------|
| `pt1` | Coin supérieur gauche | `(x, y)` |
| `pt2` | Coin inférieur droit | `(x+w, y+h)` |
| `color` | Couleur BGR | `(0,0,255)` = rouge, `(255,0,0)` = bleu, `(0,255,0)` = vert |
| `thickness` | Épaisseur en pixels | `2` |

### `capture.read()`

```python
ret, frame = capture.read()
```

| Valeur | Signification |
|--------|---------------|
| `ret` | `True` si frame lue avec succès, `False` sinon |
| `frame` | Image capturée (numpy array) |

### `cv2.waitKey(delay)`

```python
key = cv2.waitKey(1)  # Attend 1ms
if key == 27:  # Touche ESC
    break
```

| Delay | Comportement |
|-------|--------------|
| `0` | Attend une touche infiniment |
| `1` | Attend 1ms (temps réel) |
| `30` | ~30 FPS (1000ms/30 ≈ 33ms) |

**Codes touches courants :**
- `27` = Échap (ESC)
- `ord('q')` = Touche 'q'

### `capture.release()` et `cv2.destroyAllWindows()`

```python
capture.release()           # Libère la caméra
cv2.destroyAllWindows()     # Ferme toutes les fenêtres
```

⚠️ **OBLIGATOIRE** pour éviter de bloquer la caméra !

## ROI - Region Of Interest

```python
# Extraction de la zone visage
gray_roi = gray[y:y+h, x:x+w]      # ROI en niveaux de gris
color_roi = frame[y:y+h, x:x+w]    # ROI en couleur
```

Détecter les yeux **uniquement dans la ROI du visage** est crucial :
- ✅ Plus rapide (zone plus petite)
- ✅ Moins de faux positifs
- ✅ Évite de détecter des "yeux" ailleurs dans l'image

## Analogie simple

**Le cascade = Un agent de sécurité entraîné**

Imagine un agent de sécurité qui cherche des visages dans une foule :
- **Haar Cascade** = Le carnet avec les signes distinctifs (forme du visage, yeux, nez)
- **scaleFactor** = L'agent s'éloigne/recule pour voir des visages de différentes tailles
- **minNeighbors** = L'agent veut être sûr : il faut que plusieurs indices confirment que c'est bien un visage
- **ROI** = Une fois le visage trouvé, l'agent cherche les yeux UNIQUEMENT sur le visage, pas dans toute la foule

## Pièges / Astuces

⚠️ **Toujours vérifier que le cascade est chargé :**
```python
if face_cascade.empty():
    raise IOError('Fichier XML introuvable !')
```

⚠️ **Oublier la conversion RGB→BGR (picamera2)** :
```python
# ❌ Mauvais - couleurs inversées
frame = picam2.capture_array()
cv2.imshow('Detection', frame)  # Rouge ↔ Bleu inversés !

# ✅ Bon - conversion nécessaire
frame = picam2.capture_array()
frame = cv2.cvtColor(frame, cv2.COLOR_RGB2BGR)
cv2.imshow('Detection', frame)  # Couleurs correctes
```

⚠️ **Ne pas arrêter proprement picamera2** :
```python
# ❌ Mauvais - caméra reste active
picam2.start()
# ... traitement ...
# Programme se ferme sans picam2.stop()

# ✅ Bon - libération des ressources
picam2.start()
try:
    while True:
        frame = picam2.capture_array()
        # ... traitement ...
finally:
    picam2.stop()  # Toujours arrêter proprement
```

⚠️ **Ne pas oublier `cv2.COLOR_BGR2GRAY`** - Les cascades fonctionnent en niveaux de gris uniquement

⚠️ **`scaleFactor` trade-off :**
- Trop petit (`1.01`) = Très lent, mais détecte tout
- Trop grand (`2.0`) = Rapide, mais rate les visages de taille moyenne
- Sweet spot : `1.1` à `1.3`

⚠️ **`minNeighbors` trade-off :**
- Trop bas (`1`) = Beaucoup de faux positifs
- Trop haut (`10`) = Rate des vrais visages
- Sweet spot : `3` à `6`

✅ **Redimensionner pour accélérer :**
```python
frame = cv2.resize(frame, None, fx=0.5, fy=0.5, interpolation=cv2.INTER_AREA)
```

✅ **Libérer la caméra** : Toujours appeler `capture.release()` sinon la caméra reste bloquée !

✅ **Chemin absolu pour les XML** : Utiliser `cv2.data.haarcascades + 'haarcascade_frontalface_default.xml'` pour être sûr de trouver le fichier

## Différence : Caméra vs Fichier vidéo

| Aspect | Caméra (`VideoCapture(0)`) | Fichier vidéo |
|--------|---------------------------|---------------|
| Source | En temps réel | Déjà enregistré |
| Boucle | `while True` | `while capture.isOpened()` |
| Contrôle vitesse | Naturelle | `time.sleep()` pour ralentir |
| Quitter | Touche ESC | Fin du fichier ou ESC |
