# 5.4 - Détection Temps Réel avec ORB

## 🎯 Concept

ORB (Oriented FAST and Rotated BRIEF) est un algorithme de détection et de description de points d'intérêt, utilisé pour la reconnaissance d'objets en temps réel. Contrairement aux Haar Cascades, ORB apprend les caractéristiques uniques d'un objet spécifique à partir d'une image de référence.

### Points clés
- **Détection de points clés** : ORB identifie les zones distinctives de l'image
- **Descripteurs binaires** : Description rapide et légère des points détectés
- **Matching** : Comparaison entre l'image de référence et la scène actuelle
- **Homographie** : Calcul de la transformation pour localiser l'objet

---

## ⚠️ IMPORTANT : picamera2 vs cv2.VideoCapture

### Pourquoi on utilise picamera2 sur Raspberry Pi 5 ?

Sur **Raspberry Pi 5 avec caméra CSI**, `cv2.VideoCapture(0)` **ne fonctionne PAS** à cause d'un bug avec le backend GStreamer. La solution est d'utiliser **picamera2** pour capturer, puis OpenCV pour le traitement d'image.

### Différences clés

| Aspect | cv2.VideoCapture | picamera2 |
|--------|------------------|-----------|
| **Plateforme** | PC / Webcam USB | Raspberry Pi + CSI |
| **Initialisation** | `cv2.VideoCapture(0)` | `Picamera2()` + `configure()` |
| **Capture** | `capture.read()` | `picam2.capture_array()` |
| **Format couleur** | BGR (OpenCV natif) | RGB (conversion nécessaire) |
| **Arrêt** | `capture.release()` | `picam2.stop()` |

### Conversion obligatoire RGB → BGR

```python
# picamera2 capture en RGB, OpenCV travaille en BGR
frame = picam2.capture_array()           # Format RGB
frame = cv2.cvtColor(frame, cv2.COLOR_RGB2BGR)  # Conversion BGR
```

Sans cette conversion, les couleurs seront inversées (rouge ↔ bleu) !

## 💻 Code Minimal

### Version Standard (PC/Webcam USB)

```python
import cv2
import numpy as np

# Création de l'extracteur ORB
orb = cv2.ORB_create(nfeatures=1000)

# Chargement de l'image de référence
ref_image = cv2.imread('objet_reference.jpg', 0)
kp1, des1 = orb.detectAndCompute(ref_image, None)

# Matcher BF (Brute Force)
bf = cv2.BFMatcher(cv2.NORM_HAMMING, crossCheck=True)

# Capture vidéo
capture = cv2.VideoCapture(0)

while capture.isOpened():
    ret, frame = capture.read()
    if not ret:
        break
    
    gray = cv2.cvtColor(frame, cv2.COLOR_BGR2GRAY)
    kp2, des2 = orb.detectAndCompute(gray, None)
    
    if des2 is not None:
        matches = bf.match(des1, des2)
        matches = sorted(matches, key=lambda x: x.distance)
        
        # Dessiner les matches
        result = cv2.drawMatches(ref_image, kp1, frame, kp2, 
                                 matches[:10], None, flags=2)
        cv2.imshow('ORB Matching', result)
    
    if cv2.waitKey(1) == 27:
        break

capture.release()
cv2.destroyAllWindows()
```

### Version Raspberry Pi 5 + Caméra CSI (picamera2)

```python
import cv2
import numpy as np
from picamera2 import Picamera2

# ORB et image de référence
orb = cv2.ORB_create(nfeatures=1000)
ref_image = cv2.imread('objet_reference.jpg', 0)
kp1, des1 = orb.detectAndCompute(ref_image, None)
bf = cv2.BFMatcher(cv2.NORM_HAMMING, crossCheck=True)

# Initialisation picamera2
picam2 = Picamera2()
picam2.configure(picam2.create_preview_configuration(
    main={"format": "RGB888", "size": (640, 480)}
))
picam2.start()

while True:
    # Capture et conversion
    frame = picam2.capture_array()
    frame = cv2.cvtColor(frame, cv2.COLOR_RGB2BGR)
    
    gray = cv2.cvtColor(frame, cv2.COLOR_BGR2GRAY)
    kp2, des2 = orb.detectAndCompute(gray, None)
    
    if des2 is not None:
        matches = bf.match(des1, des2)
        matches = sorted(matches, key=lambda x: x.distance)
        result = cv2.drawMatches(ref_image, kp1, frame, kp2,
                                 matches[:10], None, flags=2)
        cv2.imshow('ORB Matching', result)
    
    if cv2.waitKey(1) == 27:
        break

picam2.stop()
cv2.destroyAllWindows()
```

## 🔧 Fonction détaillée

### `cv2.ORB_create()`
```python
orb = cv2.ORB_create(nfeatures=1000)
```
- **nfeatures** : Nombre maximum de points clés à détecter
- **Retour** : Objet ORB configuré

### `orb.detectAndCompute()`
```python
kp, des = orb.detectAndCompute(image, None)
```
- **image** : Image en niveaux de gris
- **Retour** : kp = keypoints, des = descripteurs

### `cv2.BFMatcher()`
```python
bf = cv2.BFMatcher(cv2.NORM_HAMMING, crossCheck=True)
```
- **NORM_HAMMING** : Distance adaptée aux descripteurs binaires ORB
- **crossCheck=True** : Matching bidirectionnel (plus précis)

### `Picamera2()` (Raspberry Pi)
```python
from picamera2 import Picamera2
picam2 = Picamera2()
picam2.configure(picam2.create_preview_configuration(
    main={"format": "RGB888", "size": (640, 480)}
))
picam2.start()
```

## 🎨 Analogie Simple

Imagine un détective examinant une scène de crime :
- Il a une **photo du suspect** (image de référence)
- Il cherche des **points caractéristiques** (tache de naissance, cicatrice)
- Il **compare** avec les personnes présentes
- Il établit un **score de correspondance** pour chaque personne

ORB fait pareil avec les images :
- Points d'intérêt = coins, textures uniques
- Descripteurs = "empreinte digitale" du point
- Matching = comparaison des empreintes

## ⚠️ Pièges et Astuces

### ❌ Erreurs fréquentes

**1. Oublier la conversion RGB→BGR (picamera2)**
```python
# ❌ Mauvais
frame = picam2.capture_array()
gray = cv2.cvtColor(frame, cv2.COLOR_BGR2GRAY)  # Erreur ! RGB passé pour BGR

# ✅ Bon
frame = picam2.capture_array()
frame = cv2.cvtColor(frame, cv2.COLOR_RGB2BGR)  # Conversion d'abord
gray = cv2.cvtColor(frame, cv2.COLOR_BGR2GRAY)
```

**2. Pas assez de features**
```python
# ❌ Trop peu de points pour matcher
orb = cv2.ORB_create(nfeatures=100)

# ✅ Plus de points = meilleure robustesse
orb = cv2.ORB_create(nfeatures=1000)
```

**3. Image de référence trop différente**
- ORB est sensible aux changements d'échelle et rotation (limités)
- Utiliser une image de référence représentative

### 💡 Astuces

**Filtre sur la qualité des matches**
```python
# Ne garder que les bons matches (distance < 50)
good_matches = [m for m in matches if m.distance < 50]
```

**Affichage du nombre de matches**
```python
cv2.putText(frame, f'Matches: {len(matches)}', (10, 30),
           cv2.FONT_HERSHEY_SIMPLEX, 1, (0, 255, 0), 2)
```

**Seuil de détection**
```python
# Minimum de matches pour considérer l'objet trouvé
if len(good_matches) > 20:
    print("Objet détecté !")
```

## 🔗 Comparaison ORB vs Haar Cascade

| Aspect | ORB | Haar Cascade |
|--------|-----|--------------|
| **Entraînement** | Aucun (image référence) | Modèle pré-entraîné requis |
| **Spécificité** | Objet personnalisé | Catégories génériques |
| **Rotation** | Robuste (jusqu'à 90°) | Peu robuste |
| **Échelle** | Limité | Variable (scaleFactor) |
| **Vitesse** | Rapide | Très rapide |
| **Usage** | Objet spécifique | Visages, voitures... |

## 📝 Résumé

La détection ORB en temps réel combine :
1. **Image de référence** avec descripteurs pré-calculés
2. **Capture temps réel** avec picamera2 (RPi) ou VideoCapture (PC)
3. **Conversion RGB→BGR** obligatoire pour picamera2
4. **Détection ORB** sur chaque frame
5. **Matching** et affichage des correspondances

**Point clé** : ORB est idéal pour reconnaître un objet spécifique sans entraînement préalable !
